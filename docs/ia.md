# 정보 구조 정의서 (Information Architecture Document)

제공된 AI 기반 게임 캐릭터 닉네임 생성기 PRD 내용을 바탕으로 설계된 정보 구조(IA) 문서입니다. 본 서비스는 회원가입 및 로그인 절차가 없는 비인증형 서비스(`auth-type: none`)이며, 상단 바(`navigation-type: topbar`) 중심의 유연하고 직관적인 웹 구조를 취하고 있습니다.

---

## 1. 사이트 맵 (Site Map)

비인증 기반 서비스의 특성에 맞춰 계층을 최소화하고, 사용자가 유입된 후 즉시 가치를 경험할 수 있는 단일 페이지(Single Page App) 혹은 얕은 계층 구조로 설계했습니다.

```
홈 (메인 닉네임 생성기 페이지) - /
├── 닉네임 생성기 (메인 UI 섹션)
├── 실시간 인기 닉네임 목록 섹션
└── 서비스 소개 / 이용 안내 가이드 섹션

고객 지원 / 안내 페이지
├── 이용약관 - /terms
└── 개인정보처리방침 - /privacy

```

---

## 2. 사용자 흐름 (User Flow)

사용자가 서비스에 진입하여 최종적으로 닉네임을 복사해 게임에 활용하기까지의 흐름입니다. 모든 입출력 인터페이스는 한국어로 매끄럽게 이어집니다.

```
[서비스 진입] -> [메인 페이지 노출] 
       │
       ▼
[국가/스타일 선택] -> (한국, 영어, 일본 중 택 1)[cite: 1]
       │
       ▼
[이름 길이 설정] -> (Slider 또는 Dropdown 조작: 2자~12자)[cite: 1]
       │
       ▼
[닉네임 생성 버튼 클릭] -> [OpenAI API 연동 로딩 애니메이션 노출][cite: 1]
       │
       ▼
[결과 리스트 확인] -> (5~10개의 추천 결과 화면 표시)[cite: 1]
       │
       ▼
[원하는 닉네임 클릭] -> [클립보드 즉시 복사 및 성공 토스트 메시지 알림][cite: 1]

```

## 3. 내비게이션 구조 (Navigation Structure)

상단 바(`topbar`) 형태의 내비게이션을 지원하며, 사용자의 탐색 동선을 방해하지 않도록 심플하게 유지합니다. 반응형 모바일 환경을 고려하여 레이아웃이 유연하게 변화합니다.

* **상단 내비게이션 바 (Topbar)**
* **좌측 (로고)**: 서비스 홈 이동 버튼 (텍스트/로고 마크)
* **우측 (메뉴 가기)**: 최근 생성 이력 조회 팝업 활성화 버튼, 다크모드/라이트모드 전환 스위치


* **하단 영역 (Footer)**
* 카피라이트, 이용약관(`- /terms`), 개인정보처리방침(`- /privacy`), 외부 레퍼런스 및 API 정보 표기.



---

## 4. 페이지 계층 구조 (Page Hierarchy)

모든 주요 기능은 메인 화면 안에서 컴포넌트 변형으로 동작하므로, 실제 URL 분리가 일어나는 깊이(Depth)는 매우 낮습니다.

* **1 Depth: 메인 페이지 (`/`)**
* **2 Depth: 상단 제어 필터 영역** (국가 선택 탭, 길이 슬라이더 컴포넌트)


* **2 Depth: 메인 콘텐츠 영역** (생성하기 버튼, AI 결과 리스트 그리드)


* **2 Depth: 서브 콘텐츠 영역** (최근 생성 인기 키워드 트렌드, 이용 가이드 블록)


* **1 Depth: 약관 및 규정 페이지 (`/terms`, `/privacy`)**

---

## 5. 콘텐츠 조직화 (Content Organization)

화면 내 레이아웃 배치를 유저가 시각적으로 빠르게 인지할 수 있도록 그리드 및 그룹화 형태로 나눕니다.

| 그룹 영역 | 콘텐츠 구성 요소 | 데이터 타입 / 표현 형태 |
| --- | --- | --- |
| **Hero / Title** | 서비스 타이틀 및 한 줄 설명 | 텍스트 (H1, H2 태그 적용으로 SEO 최적화) |
| **Filter Block** | 국적 스타일 설정, 글자 수 제한 설정

 | 라디오 버튼 그룹 / 슬라이더 바 (2~12자 조절 가능)

 |
| **Action Block** | AI 생성 요청 버튼

 | 대형 버튼 (로딩 시 스피너 인디케이터 처리)

 |
| **Result Block** | 생성된 닉네임 카드 리스트

 | 그리드 형태 카드 뷰 (각 카드 내 복사 아이콘 내장)

 |

## 6. 인터랙션 패턴 (Interaction Patterns)

유연한 반응형 디자인과 웹 접근성을 보장하는 인터랙션 표준 양식입니다.

* **입력 양식 피드백**: 슬라이더로 글자 수를 조작할 때, 우측에 현재 설정된 숫자가 실시간 숫자로 표시됩니다 (예: `5글자`).
* **로딩 상태(Loading State)**: AI 통신 중일 때 버튼은 비활성화(Disabled)되며, 결과 뷰 영역에는 스켈레톤 UI(Skeleton UI)가 돌아가 사용자 이탈을 방지합니다.
* **동적 알림 (Toast Message)**: 생성된 닉네임을 클릭하면 화면 중앙 하단에 `"닉네임이 클립보드에 복사되었습니다."`라는 안내 팝업이 2초간 노출된 후 자연스럽게 사라집니다.


* **웹 접근성 (Accessibility)**: 모든 버튼과 인터랙션 요소는 키보드 `Tab` 키로 포커스 이동이 가능하며, 스크린 리더용 아리아 레이블(`aria-label`)을 선언하여 전연령층이 쉽게 사용할 수 있도록 보장합니다.

## 7. URL 구조 (URL Structure)

비인증 방식과 단일 서비스 성격을 살려, 단순하고 명확하며 검색 엔진(SEO)에 최적화된 URL 디렉터리 구조를 차용합니다. Next.js App Router의 라우팅 시스템에 맞춤 설계되었습니다.

| URL 경로 | 대응 페이지 / 설명 | SEO 타깃 키워드 예시 |
| --- | --- | --- |
| **`/`** | 메인 서비스 (닉네임 생성기 화면) | 게임 닉네임 생성기, AI 아이디 추천, 닉네임 짓기 |
| **`/terms`** | 서비스 이용약관 페이지 | 이용약관 |
| **`/privacy`** | 개인정보처리방침 페이지 | 개인정보처리방침 |

> **SEO Best Practices 적용**:
> 메인 페이지(`/`) 로딩 시, Head 태그 영역에 `og:title`, `og:description` 및 구조화된 데이터(Structured Data)를 주입하여 네이버, 구글 등 검색 엔진 검색 봇이 사이트 성격을 명확하게 크롤링할 수 있도록 설계합니다.

## 8. 컴포넌트 계층 구조 (Component Hierarchy)

Next.js 환경에서 재사용 가능한 원자 단위를 결합하여 구성한 컴포넌트 아키텍처입니다.

```
Layout (Root Layout)
└── NavigationBar (Topbar - 브랜드 로고, 유틸 아이콘 포함)
└── MainContainer (중앙 정렬 레이아웃 배포 블록)
    ├── HeaderSection (SEO 친화적 타이틀 헤드)
    ├── GeneratorForm (필터 제어 상태 관리 컴포넌트)
    │   ├── StyleSelector (한국/영어/일본 라디오 버튼 그룹)[cite: 1]
    │   ├── LengthSlider (2자 ~ 12자 조절 슬라이더)[cite: 1]
    │   └── SubmitButton (OpenAI API 호출용 트리거 버튼)[cite: 1]
    ├── ResultDisplay (결과값 매핑 섹션)
    │   ├── SkeletonCardList (로딩 중 임시 배치 카드)
    │   └── NicknameCard (생성된 결과 텍스트 카드 + 복사 핸들러 포함)[cite: 1]
    └── InfoSection (하단 이용 꿀팁 및 가이드 컴포넌트)
└── Footer (카피라이트 및 법적 약관 링크 링크 목록)

```

이와 같은 유기적인 정보 구조 설계를 통해 저연령층부터 고령층 게이머까지 기기(모바일/데스크톱)에 구애받지 않고 가장 부드러운 한국어 UX로 AI 기반 서비스를 경험할 수 있게 됩니다.
