# AI 기반 게임 닉네임 생성기 UI/UX 디자인 가이드북

본 가이드는 앞서 정의된 제품 요구사항(PRD), 정보 구조(IA), 유스케이스를 기반으로, 전연령층 게이머가 즐겁고 명확하게 사용할 수 있도록 설계된 UI/UX 디자인 가이드라인입니다. 본 가이드는 **유쾌하고 활기찬(Playful)** 감성을 유지하면서도, 시각적인 **깔끔함**을 잃지 않는 것을 핵심 원칙으로 삼습니다.

---

## 1. 디자인 시스템 개요 (Design System Overview)

### 디자인 원칙 (Design Principles)

* **Playful & Clean (유쾌함과 깔끔함의 조화)**: 게임의 즐거운 감성을 담기 위해 둥근 코너(Rounded corners), 부드러운 애니메이션, 통통 튀는 컴포넌트 인터랙션을 활용하되, 여백(White space)을 충분히 확보하여 가독성을 높이고 깔끔한 인상을 유지합니다.
* **직관적인 한국어 UI**: 전연령층이 사용하는 서비스인 만큼 모호한 아이콘 단독 표기를 지양하고, 명확한 한국어 텍스트와 레이블을 결합하여 접근성을 극대화합니다.


* **일관된 그리드 및 반응형 동선**: 데스크톱부터 모바일 인앱 브라우저까지 어떤 환경에서도 레이아웃이 깨지지 않도록 유연한 그리드 시스템을 적용합니다.

## 2. Tailwind CSS 컬러 팔레트 (Color Palette)

주어진 유사색(Analogous) 매칭 스키마에 따라 메인 브랜드 컬러인 Sky Blue (#0099ff)를 중심으로 푸른색 계열의 서포트 컬러를 설계했습니다. 유쾌한 무드를 살리기 위해 채도가 높은 색상을 포인트로 기용하고, 깔끔함을 위해 뉴트럴 톤은 차가운 회색(Cool Gray) 계열을 배치했습니다.

| 분류 | Tailwind 명칭 | 컬러 코드 | Rationale (선정 사유) |
| --- | --- | --- | --- |
| **Primary (주요색)** | `primary-500` | `#0099ff` | 브랜드 메인 컬러. 청량하고 신뢰감을 주는 게임 허브 느낌의 Sky Blue. |
|  | `primary-600` | `#007acc` | 버튼 Hover 상태나 강조 텍스트용 딥 블루. |
|  | `primary-100` | `#e5f5ff` | 활성화된 탭 배경이나 가벼운 하이라이트 영역. |
| **Secondary (보조색)** | `secondary-500` | `#00ccff` | 유사색 스키마에 맞춘 시안(Cyan) 블루. 활기찬 에너지를 부여할 때 사용. |
|  | `secondary-100` | `#e6faff` | 서브 카드 배경색 및 미세한 영역 분할용. |
| **Accent (강조색)** | `accent-500` | `#3344ff` | 인디고 블루. 강력한 호출 버튼(CTA) 또는 주목해야 할 알림 배지용. |
| **Neutral (무채색)** | `neutral-900` | `#1e293b` | 주 텍스트 (Slate-800). 완전한 검은색보다 눈이 편안하고 깔끔함. |

|  | `neutral-100` | `#f1f5f9` | 전체 메인 배경색 (Slate-100). 깨끗하고 정돈된 화면 제공. |
|  | `neutral-0` | `#ffffff` | 컨테이너 카드 배경 및 내부 컴포넌트 기본 바탕색. |

## 3. 페이지별 상세 구현 가이드 (Page Implementations)

본 서비스는 핵심 흐름이 하나의 단일 화면에서 이루어지는 단일 페이지 애플리케이션(SPA) 구조를 취하고 있습니다. 루트 경로(`/`)를 중심으로 세부 컴포넌트가 그리드 시스템 위에 유기적으로 배치됩니다.

### 메인 생성기 페이지 (루트 경로: `/`)

#### 1) 페이지의 핵심 목적

사용자가 복잡한 탐색 없이 접속하자마자 '스타일 선택 ➡️ 글자 수 지정 ➡️ AI 생성 ➡️ 클릭 복사'의 전 과정을 한눈에 인지하고 스트레스 없이 게임 닉네임을 획득하도록 돕는 것입니다.

#### 2) 레이아웃 구조 (Grid System)

* **Desktop & Wide**: 상단 히어로 타이틀 배너 아래, 12컬럼 그리드 시스템을 사용하여 `4:8` 비율의 투 칼럼(Two-column) 레이아웃을 구성합니다.


* Left(4 Rails): 조작 필터 패널 (스타일, 길이 제어)


* Right(8 Rails): AI 결과 출력 리스트 및 트렌드 뷰 영역




* **Tablet & Mobile**: 12컬럼 전체를 사용하는 싱글 칼럼(One-column) 레이아웃으로 동적 전환되며, 상단에서 하단으로 흐르는 자연스러운 시선 이동을 유도합니다.



#### 3) 핵심 구성 요소 및 UI 텍스트 명시

* **히어로 타이틀 섹션 (Hero Section)**
* 비주얼 일러스트: 게임 감성을 자극하는 그래픽 배치
* *URL*: `[https://picsum.photos/id/180/1200/400](https://picsum.photos/id/180/1200/400)` (깔끔하고 시원한 게임 배경 연상 이미지)


* 타이틀 메인 카피: `대체 불가능한 나만의 게임 아이디` (H1, `neutral-900`, 폰트 두께 Bold)
* 서브 카피: `인공지능이 추천하는 트렌디하고 고유한 닉네임을 지금 바로 만나보세요.` (`neutral-400`)


* **조작 필터 패널 컴포넌트 (Generator Form)**
* **스타일 선택 (라디오 그룹)**: `국가별 스타일 선택`

* 선택지 버튼 텍스트: `🇰🇷 한국식`, `🇺🇸 영어식`, `🇯🇵 일본식`

* UX 디자인: 선택 시 `primary-100` 배경과 함께 `primary-500` 테두리가 두꺼워지며 통통 튀는 바운스 애니메이션 가미.


* **이름 길이 설정 (슬라이더 및 수치 노출)**: `닉네임 글자 수 지정`

* 레인지 슬라이더 바 (최소 2자 ~ 최대 12자 제약 적용)


* 현재 선택 수치 표기 우측 배치: `[ 4글자 ]` 형태의 강조 볼드 박스 노출.




* **생성 트리거**: `AI 닉네임 생성하기` 대형 버튼


* 디자인: `bg-primary-500` 바탕에 흰색 볼드 텍스트. `rounded-full`을 주어 Playful한 조약돌 느낌 선사.




* **결과 리스트 뷰 영역 (Result Display)**
* **초기 상태 안내**: `"원하는 조건을 선택한 후 생성 버튼을 눌러주세요!"` 텍스트와 함께 친근한 대기 일러스트 컴포넌트 배치.
* *URL*: `[https://picsum.photos/id/60/600/400](https://picsum.photos/id/60/600/400)` (노트북과 데스크 환경의 정돈된 이미지)


* **로딩 상태**: 스켈레톤 카드 5개가 깜빡이는 가벼운 펄스(Pulse) 애니메이션 작동.


* **결과 카드 (Grid)**: `tailwindcss grid-cols-1 md:grid-cols-2 gap-4` 구문을 기반으로 격자형 배치.


* 카드 내부 텍스트: 생성된 닉네임 문자열 (예: `"새벽의검사"`) 표기.


* 하단 유틸 안내문: `💡 클릭하면 즉시 복사됩니다!` 소형 캡션 텍스트 추가.

## 4. 공통 레이아웃 컴포넌트 (Layout Components)

### 1) 상단 바 (Topbar)

* **적용 경로**: 모든 루트 및 서브 경로 공통 (`/`, `/terms`, `/privacy`)


* **핵심 구성 요소**:
* **좌측**: 브랜드 로고 및 텍스트 (`🎮 닉네임.AI`) - 클릭 시 메인으로 리다이렉트.
* **우측**: 다크모드 토글 스위치, 서비스 안내 팝업 버튼.


* **반응형 대응 (Responsive Behavior)**:
* `desktop` 이상: 메뉴 글자가 텍스트 형태로 모두 노출.
* `mobile` 및 `tablet`: 텍스트를 숨기고 직관적인 미니멀 아이콘 아이템으로 축소 배치하여 상단바 높이를 슬림하게 유지 (깔끔함 강조).



### 2) 하단 바 (Footer)

* **적용 경로**: 전역 공통 적용
* **핵심 구성 요소**:
* 법적 고지 링크: `이용약관`, `개인정보처리방침` (클릭 시 해당 서브 라우트로 이동).


* 카피라이트: `© 2026 닉네임.AI. All rights reserved.`


* **반응형 대응**: 데스크톱에서는 좌우 배치(Flex row), 모바일에서는 중앙 정렬 세로 배치(Flex col)로 유연하게 스택 구조 변형.

## 5. 인터랙션 패턴 (Interaction Patterns)

유쾌하고 깔끔한 무드를 완벽히 구현하기 위한 시각적 피드백 약속입니다.

* **클릭 및 호버 효과 (Hover/Active State)**
* 모든 클릭 가능한 카드와 버튼은 `hover:-translate-y-1 hover:shadow-lg transition-all duration-300` 속성을 부여합니다. 마우스를 올리면 카드가 위로 부드럽게 10px 들리며 그림자가 커져 마치 살아 움직이는 듯한 'Playful' 감성을 전달합니다.


* **클립보드 복사 완료 피드백 (Toast)**
* 사용자가 닉네임 카드를 탭하면, 카드가 순간적으로 안쪽으로 들어갔다 나오는 스케일 애니메이션(`scale-95`)이 작동합니다.


* 이와 동시에 화면 중앙 하단에 `bg-neutral-900 text-white rounded-xl px-6 py-3` 구조의 깔끔한 토스트창이 솟아오릅니다.
* **내부 메시지**: `🎉 성공! "새벽의검사" 가 클립보드에 복사되었습니다.`



* **동적 스켈레톤 로딩 (Skeleton Loading)**
* AI가 닉네임을 연산하는 동안 멈춘 화면을 보여주지 않고, 회색빛의 둥근 모서리 블록들이 부드럽게 밝아졌다 어두워지는 흐름을 유지하여 시스템이 활발히 작동 중임을 인지시킵니다.

## 6. 중단점 기준 (Breakpoints)

제공된 사양 명세에 맞춰 Tailwind CSS 설정을 아래와 같이 커스텀 바인딩하여 뷰포트 분기를 제어합니다. 단일 레이아웃 컴포넌트들이 미디어 쿼리 흐름에 맞춰 완벽히 그리드 수치를 재계산하도록 강제합니다.

```scss
// 시스템 적용 중단점 규칙 (Tailwind 커스텀 세팅 동기화)
$breakpoints: (
  'mobile': 320px,   // 초소형 스마트폰 및 가로 폭이 좁은 웹뷰 대응 시작점
  'tablet': 768px,   // 패드 및 모바일 가로 모드 화면 전환 기준점
  'desktop': 1024px, // 일반 노트북 및 랩톱 모니터 최적화 배치 시점 (4:8 그리드 활성화)
  'wide': 1440px     // 대형 모니터 환경, 최대 콘텐츠 가로폭 1200px 제한 적용 영역
);

```

### 중단점별 그리드 레이아웃 변형 요약

* **`mobile` (~767px)**: 모든 컴포넌트가 단일 가로폭 블록(`w-full`)으로 세로 적층됩니다. 여백은 패딩 기준 `p-4`로 조절하여 좁은 화면을 넓게 쓰도록 유도합니다.


* **`tablet` (768px~1023px)**: 옵션 조작 패널이 상단에 가로로 길게 배치되고, 결과 리스트 카드가 `grid-cols-2` 형태로 나열되기 시작합니다.


* **`desktop` 이상 (1024px~)**: 완성형 사이드바-메인 그리드 비율(`4:8`)이 완전하게 유지되며 최상의 레이아웃 대칭 안정감을 제공합니다.
