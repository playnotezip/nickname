# 유스케이스 정의서 (Use Case Document) - AI 기반 게임 캐릭터 닉네임 생성기

본 문서는 앞서 정의된 제품 요구사항 정의서(PRD) 및 정보 구조(IA)를 바탕으로, 비인증 기반(`auth-type: none`) AI 게임 닉네임 생성기 서비스의 핵심 유스케이스를 상세히 정의합니다.

---

## 1. 액터 정의 (Actor Definitions & Comprehensive Actor Definitions)

본 서비스는 별도의 회원가입 및 로그인 과정이 없으므로 시스템과 상호작용하는 액터의 구조가 단순하며, 명확히 정의된 역할을 수행합니다.

| 액터명 | 분류 | 정의 및 설명 |
| --- | --- | --- |
| **일반 사용자 (User)** | 주 액터 (Primary Actor) | - 웹 또는 모바일 환경을 통해 서비스에 접속하는 모든 게이머.<br>

<br>- 로그인 없이 닉네임 생성 조건을 설정하고 결과를 복사할 수 있음. |
| **OpenAI API 시스템** | 부 액터 (Secondary Actor) | - 사용자가 설정한 입력값(국가 스타일, 글자 수)을 프롬프트 형태로 전달받아 자연어 처리 기반의 고유한 닉네임을 실시간으로 생성하여 반환함. |
| **닉네임 저장소 (Database)** | 부 액터 (Secondary Actor) | - 생성된 닉네임의 이력, 통계 정보, 또는 비로그인 세션 기반의 임시 북마크 내역 등을 기록하는 시스템 스토리지. |

## 2. 상세 유스케이스 시나리오 (Detailed Use Case Scenarios)

### [UC-01] AI 기반 맞춤형 게임 닉네임 생성 및 복사

* **유스케이스명**: AI 기반 맞춤형 게임 닉네임 생성 및 복사
* **액터**: 일반 사용자(주), OpenAI API 시스템(부), 닉네임 저장소(부)
* **설명**: 사용자가 원하는 국가 스타일과 글자 수 길이를 지정하여 AI가 생성한 창의적인 닉네임 목록을 제공받고, 마음에 드는 항목을 원클릭으로 클립보드에 복사하는 흐름입니다.

---

## 3. 사전 조건 및 사후 조건 (Preconditions and Postconditions)

* **사전 조건 (Preconditions)**
1. 사용자가 인터넷에 연결된 기기(Web, iOS, Android, Desktop 등)를 통해 서비스 URL(`/`)에 정상적으로 접속해 있어야 합니다.
2. 시스템과 OpenAI API 서버 간의 네트워크 통신이 가능한 상태여야 합니다.


* **사후 조건 (Postconditions)**
1. **정상 완료 시**: 생성된 닉네임 데이터가 화면에 출력되고, 사용자가 선택한 닉네임이 클립보드에 복사됩니다. 생성 이력 및 통계 데이터가 데이터베이스(Database)에 누적됩니다.
2. **오류 발생 시**: 시스템이 오류 메시지나 스켈레톤 UI 상태를 안전하게 해제하고, 사용자에게 명확한 한국어 안내 및 재시도 버튼을 제공하여 시스템 일관성을 유지합니다.



---

## 4. 메인 흐름 및 이벤트 프로세스 (Main Steps and Flow of Events)

사용자가 오류 없이 가장 표준적인 경로로 서비스를 이용하는 기본 흐름(Happy Path)입니다.

1. **[사용자]** 서비스 메인 페이지(`/`)에 진입합니다.
2. **[시스템]** 한국어 인터페이스로 구성된 조건 설정 화면(국가별 스타일, 이름 길이 슬라이더)과 최근 인기 닉네임 뷰를 화면에 노출합니다.
3. **[사용자]** '국가별 스타일'에서 하나(예: 일본식)를 선택하고, '이름 길이 설정' 슬라이더를 조작하여 원하는 글자 수(예: 4글자)를 지정합니다.
4. **[사용자]** `AI 닉네임 생성하기` 버튼을 클릭합니다.
5. **[시스템]** 생성 버튼을 비활성화(Disabled) 상태로 전환하고, 결과 출력 영역에 스켈레톤 UI 및 로딩 인디케이터를 표시합니다.
6. **[시스템]** Next.js 백엔드를 통해 사용자의 선택값(일본식, 4글자)을 매핑한 프롬프트를 구성하여 [OpenAI API 시스템]에 요청을 전달합니다.
7. **[OpenAI API 시스템]** 조건에 부합하는 고유 닉네임 5~10개를 생성하여 시스템에 반환합니다.
8. **[시스템]** 수신된 닉네임 목록을 정렬하여 화면의 결과 리스트 그리드에 한국어로 부드럽게 노출하고, [닉네임 저장소]에 생성 로그를 기록합니다.
9. **[사용자]** 제안된 목록 중 마음에 드는 닉네임 카드를 클릭합니다.
10. **[시스템]** 디바이스의 클립보드 API를 호출하여 해당 닉네임을 복사하고, 화면 중앙 하단에 `"닉네임이 클립보드에 복사되었습니다."`라는 토스트 알림을 2초간 표시합니다.

## 5. 대체 흐름 및 예외 처리 (Alternative Flows, Edge Cases and Exception Handling)

네트워크 단절, 입력값 한계, API 장애 등 다양한 비정상 상황에 대처하기 위한 흐름입니다.

### [AF-01] 외부 API 타임아웃 및 장애 (Exception)

* **조건**: 6단계 또는 7단계에서 OpenAI API 시스템의 응답이 10초 이상 지연되거나 `500 Internal Server Error` 등의 에러가 반환되는 경우.
* **처리 절차**:
1. 시스템은 로딩 상태를 해제하고 결과 영역에 에러 컴포넌트를 표시합니다.
2. 출력 메시지: `"AI 서버와의 통신이 원활하지 않습니다. 잠시 후 다시 시도해 주세요."` (한국어 명시)
3. `다시 시도하기` 버튼을 제공하여 사용자가 필터 설정을 유지한 채 즉시 재요청할 수 있도록 유도합니다.



### [EC-01] 클립보드 권한 거부 및 미지원 환경 (Edge Case)

* **조건**: 구형 브라우저 환경이거나 인앱 브라우저 보안 정책으로 인해 `navigator.clipboard` API 호출이 실패하는 경우.
* **처리 절차**:
1. 시스템은 예외(Catch)를 감지하고 토스트 알림 대신 모달 팝업을 띄웁니다.
2. 출력 메시지: `"자동 복사가 지원되지 않는 환경입니다. 아래 텍스트를 길게 눌러 직접 복사해 주세요."`
3. 팝업 내에 사용자가 수동으로 드래그 앤 드롭하여 복사할 수 있는 텍스트 입력창(Input Block)을 동적으로 렌더링합니다.



### [EC-02] 비정상적 반복 요청 및 디도스(DDoS) 방지 (Alternative)

* **조건**: 사용자가 닉네임 생성 버튼을 1초 내에 수십 번 연타하거나 매크로 프로그램을 사용하는 경우.
* **처리 절차**:
1. 프론트엔드 레벨에서 디바운스(Debounce) 처리를 기본 적용합니다.
2. Next.js 미들웨어 레벨에서 IP 기반 Rate Limiting(예: 1분당 최대 10회 생성 제안 제한)을 수행합니다.
3. 한도 초과 시 버튼 하단에 `"과도한 요청이 감지되었습니다. 1분 후 다시 이용해 주세요."`라는 경고 메시지를 표시하고 버튼을 1분간 잠금 처리합니다.

## 6. 비즈니스 규칙 및 제약 조건 (Business Rules and Constraints)

1. **입출력 언어 제약**: 사용자가 마주하는 UI 문구, 플레이스홀더, 알림 텍스트는 물론이고, AI가 생성하여 출력하는 닉네임의 결과 설명이나 카테고리 명칭은 모두 **한국어**로 정교하게 포맷팅되어야 합니다.
2. **글자 수 바운더리 정책**: 닉네임 길이는 반드시 **최소 2자에서 최대 12자** 사이의 정수 구조로 제약합니다. 프론트엔드 UI(슬라이더)에서 원천 차단할 뿐 아니라 API 요청 유효성 검증(Validation) 단계에서도 이를 엄격히 체크합니다.
3. **인증 불필요 정책**: 전연령대의 접근 편의성을 위해 로그인, 회원가입, OAuth 연동을 요구하지 않는 전면 개방 구조를 유지해야 합니다.

## 7. 사용자 인터페이스 및 UX 고려사항 (User Interface Considerations)

* **반응형 레이아웃 (Responsive Design)**
* 데스크톱 환경에서는 필터 설정 영역과 결과 그리드가 가로 배열(Split Layout)로 조화를 이루게 배치합니다.
* 모바일/iOS/Android 웹뷰 환경에서는 상단에 필터 설정, 하단에 결과 리스트가 오는 세로 배치(Single Column Layout)로 유연하게 전환됩니다.


* **접근성 (Accessibility)**
* 전연령층을 고려하여 텍스트와 배경의 명도 대비(Color Contrast)를 최소 4.5:1 이상으로 유지합니다.
* 버튼의 터치 영역 크기를 최소 44x44px 이상으로 확보하여 모바일 기기에서의 오클릭을 방지합니다.

## 8. 데이터 요구사항 및 데이터 흐름 (Data Requirements and Data Flow)

### 데이터 요구사항

* **입력 데이터**: `style` (string: ko | en | ja), `length` (integer: 2 ~ 12)
* **출력 데이터**: `nicknames` (array of strings: 생성된 닉네임 문자열 리스트), `createdAt` (datetime)

### 데이터 흐름 (Data Flow)

```
[사용자 입력: 스타일/길이] 
       │ (HTTP POST Request / 클라이언트 사이드 변수 전달)
       ▼
[Next.js API Route (BFF)] ─── (System Prompt + User Input 결합) ───> [OpenAI API]
       │                                                                  │
       │ <─── (JSON Response: { nicknames: [...] }) ──────────────────────┘
       ▼
[Next.js API Route (BFF)] ─── (로그 데이터 비동기 삽입) ───> [Database]
       │
       ▼
[클라이언트 리액트 컴포넌트] (상태 업데이트 및 한국어 카드 UI 렌더링)

```

## 9. 보안 및 개인정보 보호 고려사항 (Security and Privacy Considerations)

1. **개인정보 수집 제로**: 비인증 구조(`auth-type: none`) 서비스 특성에 부합하도록 이메일, 이름, 생년월일, 디바이스 고유 식별값(UUID) 등 어떠한 개인식별정보(PII)도 서버 데이터베이스에 저장하거나 로그로 수집하지 않습니다.
2. **API 키 은닉**: OpenAI API Secret Key는 절대 클라이언트 브라우저 코드에 노출되어서는 안 됩니다. Next.js의 환경 변수(`.env.local`)에 안전하게 보관하고, 무조건 서버 사이드(API Routes) 환경에서만 호출하도록 강제합니다.
3. **비속어 및 유해어 필터링**: OpenAI 프롬프트 지시어에 사전에 금칙어(욕설, 성적 표현, 정치적 혐오 발언 등) 생성 방지 가이드라인을 주입하며, 결과 반환 직전 2차적으로 한국어 표준 비속어 사전 필터링 로직을 Next.js 서버 내부에서 실행하여 전연령층 게이머가 안전하게 이용할 수 있도록 방어막을 구축합니다.
